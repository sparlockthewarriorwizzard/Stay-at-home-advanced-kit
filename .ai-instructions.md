# AI Instructions / Context Rules

**READ THIS FIRST.** This file defines the immutable technical stack and coding standards for this project. Adhere to these rules strictly to avoid hallucinations or incompatible code.

## üõ† Tech Stack (IMMUTABLE)

- **Language:** TypeScript (Strict mode).

## üß± Project Structure
- `src/screens`: **Full-page views only**. Must be registered in `RootNavigator.tsx`.
- `src/components`: **Reusable UI elements** (Buttons, Cards, Inputs).
- `src/services`: **Singleton logic** (API wrappers, Auth, Payments). No UI code here.
- `src/types`: **Shared TypeScript definitions**.
- `src/navigation`: **Navigators only**.

## üìù Coding Standards
1. **Functional Components Only**: Use `React.FC` or standard function declarations. NO Class Components.
2. **Hooks**: Use custom hooks (in `src/hooks`) for complex logic separation.
3. **Styles**:
   - Use `StyleSheet.create()`.
   - NO inline styles for complex objects (performance).
   - Use `src/theme` tokens if available.
4. **Environment Variables**:
   - JAMAIS import generic `process.env`.
   - ALWAYS use: `import { KEY_NAME } from '@env';`
5. **Types**:
   - Explicitly type all Props and State.
   - Avoid `any`. Use `unknown` if necessary.

## ü§ñ Common Tasks Protocol
**When asked to add a Screen:**
1. Create `src/screens/NewScreen.tsx`.
2. Add it to `RootStackParamList` in `RootNavigator.tsx`.
3. Add the `<Stack.Screen />` entry in `RootNavigator`.

**When asked to fetch data:**
1. Check `src/services/Supabase.ts` for the client instance.
2. Create a specific service method (e.g., `UserService.getUser()`) rather than calling raw SQL in a component.
